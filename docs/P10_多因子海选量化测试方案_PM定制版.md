## 维度一：Polymarket 双向交割的绝对法则 (基准价结算)
为了杜绝传统量化中“扛单”带来的数据虚胖，测试的盈亏判定将完全按照 PM 的二元期权规则执行：
1. **触发锚定**：当某个【指标组合】在本周期内打分越出“上/下安全边界（阈值）”，系统立刻记录当前价为 **基准价 (Base Price)**，并全仓虚拟下注 YES 或 NO。
2. **结算锚定**：根据您要求的 PM 规则，系统严格平移到该周期的 **固定结算时刻**（如：1H 判定该小时的整点收盘，1D 判定美东午夜 EST Midnight）。
3. **双向判定规则**：
   * **做多信号 (BUY YES)**：结算价 $\ge$ 基准价 = Win 获胜。反之 = Loss 亏损。
   * **做空信号 (BUY NO)**：结算价 $<$ 基准价 = Win 获胜。反之 = Loss 亏损。
   * **横盘不出手区间**：在安全阈值中间的分数，定性为横盘死区，**双眼紧闭，全盘观望 (Observe)**，保留现金，不计任何发车。

## 维度三：动态横盘阈值扫描 (Grid Search on Thresholds)
您敏锐地指出了“缠论的横盘范围是 $\pm35$，你现在说因子的 $\pm60$ 横盘是不严谨的”。
完全正确！60 只是我拍脑袋定的主观极限值，但真正赚钱的数据必须要靠数据自己跑出来！所以，本方案将引入 **矩阵阈值穷举扫描**。

对于每一个指标和组合，脚本将分别用以下 **4 个渐进级别的“阈值门槛”** 跑一遍 300天（超10万根 K 线）的历史数据：

1. **阈值 $\pm 40$ 扫描 (高频模式)**：放宽做多/做空条件，缩小横盘观望区 $(-40 \sim +40)$。这种模式下会极高频地下单，测试“只要稍微有点迹象就压单”的胜率。
2. **阈值 $\pm 50$ 扫描 (中性模式)**：标准横盘观望区 $(-50 \sim +50)$。
3. **阈值 $\pm 60$ 扫描 (狙击模式)**：这就是之前用的硬标准。横盘观望区拉长到 $(-60 \sim +60)$。只有真正的爆量/极端超买超卖才会开枪。
4. **阈值 $\pm 70$ 扫描 (核武模式)**：将横盘观望区扩大到极致 $(-70 \sim +70)$。这种模式 300 天也许只开 20 枪，我们要看看这种绝对苛刻的条件能不能达到类似 80%~90% 的逆天胜率。
