> **修订版本**：1.0 (针对 P10 升级后暴露的前端语义分裂与丑陋排版问题)
> **核心痛点**：当前 Polymarket 投注指南展开后呈现两极分化状态：一是大片高纯度土黄褐色警告条在黑色基调中极为刺眼；二是“主行显示回调劣势、展开后建议中性观望”的逻辑精神分裂；三是缺少对引擎底层分歧的立体化文字解析（Engine Analysis）。

## 方案 1：针对“精神分裂”的后端数据流漏传补救与词表统一
这一漏洞的核心在于数据跨层级传输时发生了“丢包”。

### 1-A. `prediction.py` 数据强力透传
*   **现象溯源**：`chanlun_service` 其实早已正确根据时间框架结算出了极端的加权总分（如 `composite_raw` = -62.5，强烈看空）。但在把它喂给 `prediction.py` 进行盘口组装时，`prediction.py` 未将该真实分数塞入最终弹给前端的 JSON 内。这导致服务层接收到空值，被迫 fallback 兜底触发了“50 中性”。
*   **动作**：强制在 `app/engines/prediction.py` 中将传入的 `raw_score` (`composite_raw`) 原封不动挂载到 `compositeWinRate`。
*   **预期**：前后端的底层分值达成 $100\%$ 同频！

### 1-B. `polymarket_service.py` 的判决树强行坍缩合并
*   **现象溯源**：当前代码依然存在老旧的两套说辞网络。例如：如果分数不极端则 `score_level` 被标为“中性”，如果盘口看跌则 `spot_momentum` 又被标为“回调压力”。
*   **动作**：重写这几者的逻辑关联。**必须坍缩为一套逻辑树！**
    *   只要最终算出来的预测方向是 **down** 且分数抵达预警阀值，哪怕级别不够“极佳”，评语树必须强制输出诸如“看空抵抗”、“回调下行”等词汇，**任何看跌做空的环境下，绝不允许打出“中性”这两个字来扰乱心智！**

## 方案 2：前端 UI 颠覆性重构 (The Split-Pane Diagnostic Card)

用户已经直接在主行获得了极其紧凑高效的一行判决（例如：*良好 — 趋势强烈可建仓，综合80分   缠论引擎 -4分 (down) | 六因子 -6分 (down)*）。如果在下拉款内继续用大黄块来包围这句话，不仅丑陋且丧失了高级感。

### 降维视觉痛点：
我们必须用属于顶级交易终端的质感取而代之。

### 【第一版块】：引擎深度共振解析 (Engine Consensus Analysis)
*   **UI 摒弃**：彻底废除底色高达 100% 不透明度的黄/红/绿块，改用带有边框的**透明磨砂玻璃卡片** (Glassmorphism / Subtle Bordered Layout)。
*   **解析注入**：不再是读干瘪的分数。前端将动态对比 `chanlun` 与 `factor`。
    *   **例析 1（高共振）**：*结构网络与动能因子模型当前达成极高双边下行共振。多维度下杀条件已成熟，建议重仓博弈本周期波段空头。*
    *   **例析 2（强撕裂）**：*当前呈现致命的因子撕裂。长周期形态依然处于上扬通道，但短线暴跌引发动量六因子全面看跌。这种矛盾极易导致画门清算，切忌满仓！*

### 【第二版块】：微观盘面异动雷达 (Market Anomalies Tracker)
*   右侧或下方区块保留给所有的纯粹 Triggers。
*   把诸如 `[价格触碰中枢下沿]`、`[RSI极度超卖 (<25)]` 格式化为极其干练的**暗色科技子弹晶体列 (Bullet Chips)**，如同交易系统的告警灯。
