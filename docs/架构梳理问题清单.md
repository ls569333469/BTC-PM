## 已发现的问题

### 问题1：缠论引擎与6因子评分关系错误
- **现状**：导图中把6因子评分画成缠论引擎流水线的最后一步（串联）
- **实际**：缠论分析和6因子评分是**两个独立的评判系统**，应该是并行关系
  - 缠论引擎（主）：K线 → 笔 → 中枢 → 背驰 → 趋势 → **方向+目标价**
  - 6因子评分（辅）：MACD背驰 + 成交量 + 布林带 + 资金费率 + 恐惧贪婪 + RSI → **综合评分**
  - 两者在 `chanlun_service.py` 里合并 → 最终胜率
- **影响**：导图误导，逻辑不清晰
- **修复**：更新导图，画成并行两条线最后汇合

---

### 问题2：分析间隔(15min) vs 结算间隔(5min) 的设计约束
- **现状**：`auto_analysis` 每15分钟产出新预测，`auto_resolve` 每5分钟检查到期预测
- **说明**：`auto_resolve` 不需要新预测，它只是从DB查已到期的旧预测去结算，所以5分钟检查一次是合理的（确保30M预测能及时被发现到期）
- **真正的问题**：PM支持5分钟市场，但15分钟才产出一次预测，意味着无法支持15M以下的PM市场预测
- **建议**：如果未来要支持5M/15M PM市场，需要将分析间隔缩短，但要考虑外部API调用频率和性能
- **当前结论**：设计合理，暂时保持不变，记录为已知约束
