## 概述

P3 分两条主线：**部署上线**（让项目在 VPS 上 7×24 运行）和**数据驱动增强**（利用持续积累的回测数据优化系统）。

---

## P3A — VPS 部署修复 ⚙️

> 现状：已有 `deploy.sh` / `nginx.conf` / `chanlun.service`，但存在多处路径和版本问题

### 需修复的问题

| # | 问题 | 位置 | 修复 |
|---|------|------|------|
| 1 | 部署路径不一致 | `deploy.sh` 使用 `/var/www/chanlun`，但 `chanlun.service` 和 `nginx.conf` 使用 `/opt/BTC-PM` | 统一为 `/opt/BTC-PM` |
| 2 | Python 版本不存在 | `deploy.sh` 指定 `python3.14`（不存在） | 改为 `python3.12`（Ubuntu 24.04 默认） |
| 3 | DB 密码硬编码 | `deploy.sh` 生成 `.env` 时 DB 密码是 `postgres` | 从 VPS 环境变量读取或生成随机密码 |
| 4 | systemd 用户权限 | `chanlun.service` 使用 `root`，`deploy.sh` 使用 `www-data` | 统一为 `root`（简易部署） |
| 5 | CORS 未更新 | `.env` 模板中 `CORS_ORIGINS` 需要填入实际 VPS IP | 部署时自动填入 |

### 部署步骤

```
1. SSH 连接 VPS (107.172.78.150)
2. git clone https://github.com/ls569333469/BTC-PM.git /opt/BTC-PM
3. 运行修复后的 deploy.sh
4. 验证: curl http://107.172.78.150/api/health
```

### 涉及文件

| 文件 | 改动 |
|------|------|
| `deploy.sh` | 统一路径 + Python 版本 + 密码处理 |
| `.env.production` | [NEW] VPS 专用配置模板 |

## 风险提示

> [!WARNING]
> **Binance API 在美国 IP 被封锁（HTTP 451）**。VPS 位于美国 (RackNerd)，部署后所有 Binance API 调用可能失败。
> **解决方案**：使用 Binance 的备用域名 `api1.binance.com` / `api2.binance.com`，或通过非美国代理中转。

> [!IMPORTANT]
> VPS 部署后需要确保 PostgreSQL 密码不是默认值 `postgres`，且 `.env` 不被 git 追踪。
