# 第7阶段：历史回测引擎实施计划

## 目标描述
本阶段的目标是构建一个强大而独立的“历史回测引擎”（`run_historical_backtest.py`），能够跨越时间遍历数月的币安历史数据。与现有的等待实时时间流逝的“前向测试”不同，新系统将自主模拟时间的推移，在真实的历史数据帧上循环启动缠论预测引擎，运用“上帝视角”瞬间评估预测结果，并极其高效地提炼出高胜率的指标组合（“Alpha 挖掘”）。该系统完美地将币安精确的分钟级收盘价代理为 Polymarket 基准价（Base Price），从而安全彻底地绕过了 Polymarket API 的请求限制与历史查阅瓶颈。

## ⚠️ 关于“回测天数”与“新闻/基本面影响”的核心解答区

> [!NOTE]
> **关于回测天数（绝非仅仅 3 天！）**
> 之前计划中提到的“硬编码为 3 天”，**仅仅是在开发阶段用于《校准单元测试》**（因为我们线上的实盘数据库只存活了 3 天，我们用前 3 天跑一遍，是为了跟线上数据对账，证明代码没写错）。
> **在真正的实战回测中，您的执行指令将是：`python run_backtest.py --days 90`（直接回滚过去 3 个月）甚至 `--days 365`（回滚过去 1整年）！** 我们的数据通道支持无限回滚。

> [!IMPORTANT]
> **关于缺失“新闻事件影响（News/Fundamentals）”的终极解决方案**
> 这是目前纯技术面量化系统的通病：技术指标是瞎子，它不知道今晚 SEC 要发难，或者美联储要降息。在无法以极高成本获取“过去 3 个月所有的推特情绪 API 历史库”的情况下，我们通过以下两种手段实现**降维打击**：
>
> 1. **新增《异常波动/巨量插针（Anomaly）过滤器》**：
>    在缠论哲学的最高奥义中：“任何的基本面消息，最终一定会第一秒钟砸在盘面的「成交量」和「波动率」上”。
>    我们将要在回测系统里额外算入一个因子：如果在当前的 5分钟/15分钟 K线内，**成交量突然暴增超过 20周期均量的 500%**，或者**单根 K 线的实体幅度（ATR）瞬间放大 5 倍**，系统将强制打上 `[NEWS_ANOMALY]` 标签。在回测报告中，您可以一秒钟将所有的“疑似突发新闻黑天鹅阶段”剥离出去不看，或者专门去研究“黑天鹅发生后 1 小时内的反转胜率”。
>
> 2. **导入《宏观日历（Macro Calendar）》静态映射表**：
>    新闻有“突发（如战争/爆雷）”和“已知（如 CPI 数据公布、美联储 FOMC 决议）”两种。我们将会在回测脚本旁放一个极为轻量的 `macro_events_2024.csv`（记录过去几个月的关键数据公布时间，如 3月12日 20:30 CPI发布）。回测引擎在遍历到这些绝对危险的 **前 5 分钟** 时，会直接强行标记 `[MACRO_EVENT]`。这就等同于让引擎拥有了“看宏观新闻”的视力。

## 拟议变更

### 1. 离线回测主程序组
#### [NEW] `backend/app/scripts/run_backtest.py`
专为本地命令行运行的大型模拟推手程序。
* **时间流推演循环**：在下载好的币安长达数月的大数据上，将“当前观测尽头”每次向后推进 5 分钟。
* **物理防作弊调用**：强制截断历史切面以阻止引擎“偷看未来”。
* **【新增】新闻/巨量阻断器**：实时计算短期 ATR 偏离率与成交量倍数，一旦超阈值，立即标记此帧带有“疑似突发新闻冲击”属性。
* **快速开奖核对**：瞬间跨越时间轴（+ 1 小时或 + 24 小时），利用未来线核对开仓是赚是亏。

#### [NEW] `backend/app/scripts/backtest_analyzer.py`
量化分析汇总与策略解刨室。
* 接管 `run_backtest.py` 输出的上万行海量记录台账。
* **战术价值**：提炼 Alpha 矩阵。例如：*“当 `当前时间框架=4H`、且 `MACD刚刚发生底部背离=True`，且 `长天期突破综合发力分 > 60` 时，这 6 个月里的实战综合胜率（Hit Rate）究竟是多少？”*
* **【新增】剥离宏观噪音**：一键剔除所有带有 `[NEWS_ANOMALY]` 或 `[MACRO_EVENT]` 标签的爆仓样本，看看纯技术面下的理论胜率有多恐怖。
