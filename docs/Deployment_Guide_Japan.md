# 🇯🇵 BTC-PM 量化对冲系统：日本节点上云部署指南与报价单

针对本系统（Vite 前端 + FastAPI 后端 + 重度 NumPy/Chanpy 计算引擎）的特性，考虑到币安（Binance）API 的低延迟优势，**强烈推荐部署在日本东京节点（ap-northeast-1）**。

以下是为本系统专门定制的云服务器推荐方案及报价清单：

---

## 💻 1. 最低与推荐硬件配置标准

由于底层涉及 8 个时间周期的 K 线大矩阵运算以及 `TA-Lib` 编译环境，对内存和环境兼容性有硬性要求：

*   **操作系统 (OS)**：必须选 **`Ubuntu 22.04 LTS`** 或 `20.04 LTS`。（强力避坑：CentOS 或 Alpine 等系统在编译 C++ 依赖时会疯狂报错，Ubuntu 兼容性最佳）。
*   **最低配置 (小资金/个人预研)**：**2 核 CPU / 2GB 内存 / 40GB SSD**（需设置虚拟内存 Swap 防崩溃）。
*   **推荐配置 (实盘全自动/低延迟)**：**2 核 CPU / 4GB 内存 / 60GB SSD**（可常驻高性能计算阵列无压力）。

## 🏢 2. 主流海外云厂商推荐清单 (均可选东京节点)

**优势**：全线 NVMe 固态硬盘，面板极其好用，按小时计费，遇到被墙换 IP 极速（随时销毁重建）。
*   **推荐产品线**：`High Frequency Compute` (高频 CPU，对跑 Chanlun 这种单核计算密集的脚本有奇效)。
*   **预估价格**：
    *   1 核 / 2GB：**$12 / 月**（约 ¥85）
    *   2 核 / 4GB：**$24 / 月**（约 ¥170）

### 🥈 选项 B：AWS Amazon EC2 (极限网速/高可用之王)
**优势**：亚太金融结算核心，距离币安 aws-ap-northeast-1 实体机房物理距离最近（在同一个网段内 API 延迟几乎忽略不计）。
*   **推荐产品线**：`t3.small` 或 `t3.medium` 系列（平时低负载积累积分，每 5 分钟轮询计算时爆发）。
*   **预估价格**：

**优势**：非常适合跑前后端分离的系统，提供海量社区一键傻瓜式部署教程。
*   **推荐产品线**：`Basic Droplets (Premium Intel)`
*   **预估价格**：
    *   2 核 / 2GB：**$14 / 月**
    *   2 核 / 4GB：**$24 / 月**

## 🛠️ 3. 正式环境生产架构 (Production Stack)

购买服务器并安装好 Ubuntu 以后，您不必在服务器上打开 IDE（如 VSCode）。需要进行以下规范化生产部署：

1.  **代码同步**：在服务器里通过 `git clone` 拉取您今天刚同步到 GitHub 上的最新分支！
2.  **环境配置**：
    *   执行 `sudo apt-get install build-essential python3-dev` 等底层 C 插件。
    *   执行 `sudo apt-get install libta-lib0` 解决 Ta-Lib 兼容性。
3.  **后端持久化驻留 (PM2 或 Systemd)**：
    *   不能用自带终端直接跑 `uvicorn`（因为关掉 SSH 软件后端就死了）。
    *   将部署为后台长生进程：`pm2 start "python -m uvicorn app.main:app --host 0.0.0.0 --port 8000" --name btc-backend`，这样无论报错还是断网，都会自动满血复活 1 秒拉起。
4.  **前端打包流与反向代理 (Nginx)**：
    *   执行 `npm run build` 生成极致压缩的 `dist` 静态文件夹。
    *   安装 Nginx 托管该文件夹，并在 Nginx 配置中把 `http://您的域名/api/` 的流量原封不动拦截，暗中反向代理给本地的 8000 后端端口。
5.  **SSL 域名加密 (Let's Encrypt HTTPS 安全盾)**：
    *   用一条 `certbot` 指令颁发全球信任的免费绿锁 HTTPS 证书。以应对手机端访问及各种 WebSocket 的数据安全加密需求。
