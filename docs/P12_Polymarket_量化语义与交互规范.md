# P12：Polymarket 量化语义与前端交互规范白皮书

本文档确立了 Polymarket 预测模块在整个应用中的唯一标准话术库和视觉渲染规范。所有代码迭代与 UI 升级均需严格恪守本规范，彻底杜绝传统永续合约（“做多”、“平仓”、“画门”）等不专业术语的污染。

---

## 1. 核心交易指令 (PM Action Advice) 术语库

**核心准则**：Polymarket 本质上是基于概率的“有价二元期权 (Binary Options)”。我们的策略核心不是预测绝对涨跌，而是寻找**正期望值 (+EV: Expected Value)** 和 **概率边际 (Edge)**。

| 原有错误术语 (已废弃) | P12 标准量化术语 (强制启用) | 触发逻辑与说明 | 视觉规范 (Micro-Accent) |
| :--- | :--- | :--- | :--- |
| 高胜算局 | **↑ 强烈看涨** | 预测套利空间极大，胜率极高 | `text-emerald-400` |
| 盘口优势 | **↑ 偏向看涨** | 存在明显的正向概率边际 (Edge) | `text-emerald-400` |
| 盘口震荡 | **— 无利可图** | 无利可图，盘口无套利空间 | `text-gray-400` |
| 高风险局 / 建议做空 | **↓ 强烈看跌** | 当胜率极低时，这意味着目前 YES 份额被严重高估，玩家应当去买对立面的 NO 份额 | `text-rose-400` |

*(注：系统中的 `— 无利可图` 必须严格保持冷寂的灰色脱敏状态，不得附加任何正向的高亮色。)*

## 2. 长文引擎解读 (Engine Analysis) 话术基准

在展开的 `[核心诊断]` 卡片中，引擎自动生成的长文解读必须对齐**“顶级华尔街量化机构研报”**的行文风格。

### 🟩 场景 A：双引擎同向共振极好时
> “底层资产（BTC）古典结构与动能因子达到最高维度的共振。胜率底池充裕，当前 PM 盘口（YES/NO）存在极高确定性的正期望（+EV）博弈空间。”

### 🟥 场景 B：指标互斥（发生撕裂）时
> “底层资产呈现极致的引擎撕裂：缠论形态与微观因子发生背离。PM 预测市场属二元期权，切忌在此刻进行无概率优势的 50/50 盲目掷骰子，建议空仓观望。”

### ⬛ 场景 C：平庸无水花（低信度）时
> “双引擎处于未共振的低信度水域。底层资产暂无突破确认，对应 PM 盘口的 YES/NO 份额均无显著的概率偏差（Edge），当前买入份额的长期数学期望为负（-EV）。”

## 3. UI 布局与保留矩阵声明 (Layout Matrix)

1. **不可篡改的 10 列神圣矩阵**：
   多时间框架预测表及 Polymarket 投注指南的表头，必须**原汁原味地保留原有 10 列核心参数结构**。坚决禁止为了视觉“减负”而自作主张删去任意专业指标列。
   *(周期 / 盘口基准 / 目标预测 / PM 套利空间 / 核心交易指令 / 现货锚定 / 综合评分 / 缠论 / 因子 / 多空状态)*

2. **自适应拓扑扩充**：
   表头需使用 CSS Grid 的 Fractional 布局 (`1.2fr`, `1fr`) 来动态填满高分辨率屏幕的空白领域，绝对禁止使用硬编码死值导致的右侧大规模真空带。

3. **极视角·微光信号矩阵 (Micro-Accent Matrix)**：
   遵守此前已确认并生效的 `>0` (Emerald) 与 `<0` (Rose) 隔离染色逻辑。详情见开发对话历史对微光追踪机制的确立，底层卡片永远保持 `#080808` 极黑深空背景。
