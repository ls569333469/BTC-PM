# Phase 6: 基准价动态锚定与 Hybrid Oracle 数据库 (P6 实施方案)

## 🎯 业务目标 (Summary)
彻底解决 Polymarket Gamma API 隐藏 5M/15M/4H 等超短周期期权开区**基准裁判价 (priceToBeat)** 的缺陷。
通过引入**本机级别的高并发 RTDS WebSocket 持久化数据库**，结合**币安历史 K 线容灾**，将前端赌盘的基准价格像磐石一样锁死在开盘那一秒，完全重现 Polymarket 官方的前端逻辑。

## ⚠️ 注意事项与潜在风险 (Points to Note)
1. **多线程 SQLite 并发**：背景线程持续 `INSERT`，而 Web 路由的高频 `GET` 请求同时发生。需要引入 `autocommit=True` 的短连接查询，或使用 Python `queue` 来确保 DB 不会锁死 (Database is locked)。
2. **历史脱节 (断网容灾)**：当本地服务器因升级或挂机下线时，背景线程可能会漏掉诸如 `09:15:00` 这一关键定格帧。必须无缝对接 Binance `/fapi/v1/klines` (或现货) 兜底获取同一秒的 Open 价。
3. **TUN / 跨国代理拦截**：Polymarket 的 WebSocket `wss://ws-live-data...` 极度敏感。虽然代码运行在本地后端，但必须确保它顺利流经 TUN 设置，不受 HTTP 代理规则的副作用阻断。
